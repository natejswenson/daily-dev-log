## What Shipped

The daily brief took about six minutes to write, and that is a number you feel every morning.

The obvious fix looked like parallelism: split the one big generation into a fan-out and compose the cards at the same time. I designed it, ran it through a quality gate, and then measured before building anything. The measurement killed it. Concurrent calls only ran 1.44x faster at three-wide, under the bar I had set in advance, so fan-out would have added a pile of moving parts to buy almost nothing.

So I instrumented a real brief to find where the time actually goes. About 208 of the 230 seconds was a single block of model thinking; roughly 93% of the tokens were reasoning the reader never sees. The database tools were not the problem, they took 13 milliseconds.

The lever that worked was not the one I reached for first. The thinking-budget setting is silently ignored on this path, but the reasoning-effort setting is honored. Dropping the brief to low effort cut it to about 80 to 95 seconds. Then the part that matters for a coach: a blind judge scored the low-effort briefs against the old default, and the low ones won on every measure, specificity, voice, no repetition, no filler. Less thinking made the brief tighter, not worse.

Low effort had one tell. The model sometimes dropped a table row break and collapsed a markdown table into one unreadable line. So table rendering moved into code: one shared renderer, plus a repair at the save step, so every brief renders clean no matter how the model gets sampled.

It ships as an env-tunable effort setting that defaults to low, token instrumentation so the next person does not have to guess where the time went, the table fix, and the measurement probes that made the call.

## What's Next

A wider A/B across more days and a second model to lock in the quality read, and an optional pass that assembles the brief's data in code to trim the last few seconds of tool round-trips.
