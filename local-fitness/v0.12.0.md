## What Shipped

The release process got an actual shape. Work now lands on a feature branch,
merges into `dev`, and promotes to `main`, and the release tag is cut
automatically when it gets there. This landed as plumbing first, before branch
protection, so the wiring is in place and the rules come next.

Alongside that I cleared out a stack of dependency updates that had been piling
up. The web builder moved from Node 22 to Node 26, which needed an explicit
corepack install to keep the build green. On top of that came bumps to react and
its types, react-router-dom, lucide-react, vite and the vite react plugin, plus
the CI actions: `setup-uv` and `checkout`. Nothing user-facing changed; this is
the maintenance that keeps the project from rotting quietly.

## What's Next

With the branch model wired up, turning on branch protection for `dev` and
`main` is the natural follow-up.
