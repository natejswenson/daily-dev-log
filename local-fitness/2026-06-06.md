## What I Built

local-fitness is my self-hosted fitness coach: it pulls Garmin data into a
local SQLite database and lets a Claude agent write me a daily briefing. Today
was about treating it like a real piece of software instead of a personal
script.

The core idea: an agent's prompt is the part of the app you can't compile, so
you have to *score* it instead. I wrote a small eval that grades the agent's
prompt against grounded pass/fail checks — does it keep the never-fabricate
rule, translate the training-load jargon, keep the right coaching tone, wire in
saved user notes? The check I'm happiest with cross-validates the briefing
prompt against the actual output schema: every metric and tone the prompt
advertises has to be a real member of the Pydantic enums the output is
validated against. If the prompt and the schema ever drift apart — which would
silently break briefings, since the UI drops unknown fields — the build now
fails loudly instead.

On top of that I added a pytest suite covering the deterministic core (database
helpers, the user-notes store, schemas, baselines, and the agent's query
tools, all run against a seeded temp database with no network), put it behind a
whole-repo coverage gate, and wired up CI to run linting, the tests, and the
prompt scorer on every push. The first CI run actually caught a real bug: a
batch of security tests had been silently depending on my real local database
and blew up on a clean checkout, so I made them hermetic.

Finally I added a version-driven release workflow — once CI is green on the
default branch it cuts a GitHub Release for whatever version is in the project
file, but only if that version is new, so a normal merge is a no-op and bumping
the version is what ships. That shipped the first tagged release, v0.1.0. I
closed the day by rewriting the README: it read like private notes ("my
three-year history", "built around my Garmin Instinct Solar"), so I reframed it
as a shareable open-source project — added a license, badges, a privacy
section, and cross-platform setup — without overclaiming what it does.

## What's Next

Now that the prompt is scored and the core is tested, the obvious next move is
to widen coverage into the briefing-generation and chat paths (the parts that
talk to Claude), and to start treating prompt changes as releases in their own
right.
