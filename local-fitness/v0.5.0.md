## What I Built

Today was the big one: I inverted the architecture of local-fitness. It was already a Claude-powered app, but the brief generator and the chat both ran *inside* the web server. Now the server runs **no Claude inference at all** — it serves my Garmin data and the deterministic compute (recovery baselines, the CTL/ATL/TSB training-load model, training-plan grading) over REST and the Model Context Protocol, and every act of *synthesis* — the daily brief, conversational coaching, plan drafting — moves out to a client agent I talk to from Claude Desktop / Code / Mobile. The cut line that made it tractable: deterministic compute stays as code; LLM synthesis moves to the agent. The web UI becomes a fast visual glance, not a place I converse.

I shipped it in phases, each behind a gate. **Phase 1** added a Claude-free brief write gate — a single `save_brief` function that's the only path that validates and atomically writes a brief — plus a `save_brief` MCP tool and a `brief` prompt so the agent can compose one, and a `load_latest` fallback so the Today tab never goes empty. **Phase 2** reworked the React frontend into a pure viewer: out went the embedded chat, the "generate brief" buttons, the per-dashboard insight chats; in stayed every chart, takeaway, and table. **Phase 3** deleted the server-side Claude loops outright — the chat module, the `/api/chat` and `/api/brief/generate` endpoints, the `chat`/`ask` CLI commands. I also added an `ops/` launchd job so the daily brief runs as a scheduled composer (it's a separate process — the one place the server side still talks to Claude).

Rebuilding the container surfaced a yak worth shaving: Vite 8 now bundles with rolldown, whose native binding won't resolve on Alpine, and corepack's pnpm wouldn't install the Linux binding from my macOS lockfile. Fixed by building the SPA on Debian and pinning pnpm to the version that resolves it.

Then the part I'm most glad I did: I ran my own adversarial **quality-gate + siege against the live, running container and MCP** — not just the source. It earned its keep immediately. It found a **Fatal**: the `run_sql` tool's "read-only" guard was a keyword denylist, and a CTE-prefixed query with a newline after the write keyword (`WITH a AS (SELECT 1)\ndelete\nfrom ...`) slipped past it and *committed* — demonstrated live. Plus an authenticated denial-of-service (an unbounded query froze the single event-loop thread) and two input-validation gaps in the other tools. I fixed `run_sql` to be read-only at the SQLite engine itself (so writes fail regardless of phrasing) with a bounded, offloaded execution, and hardened input validation across the MCP surface — 19 new regression tests. The gate converged clean over three rounds.

Smaller polish on top: a stale-brief banner that could never clear in the evening (the container runs UTC, so it sees a phantom "tomorrow" data row — I clamp the staleness check to the viewer's real local day), turning "ask your coach" from dead text into a button that copies a ready-to-paste MCP prompt, and a full README rewrite around the agent-first / MCP story. All of it shipped as **v0.5.0**.

## What's Next

Set the container timezone to local to kill the phantom-data-row at its source (the banner handles it client-side now, but cleaner data is better), and keep leaning into the MCP as the primary surface — the goal is to reach for the coach in Claude more than the UI.
