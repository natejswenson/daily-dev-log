## Shipped

Ghostwriter drafts LinkedIn posts in your voice. Until this release, the flow opened by interviewing you: a couple of generic questions to figure out what the post should be about. Version 0.6.0 throws that out. It now opens with a menu of concrete, already-grounded ideas: pick a lane (an industry radar, one of your recent projects, or an evergreen angle), then pick a specific anchor, and that pick is the post's real subject. The radar got widened past one vendor to the whole AI industry, and a new discovery step finds your recent projects by reading Claude Code's own session logs. The interesting engineering is that last part, so let me walk it.

## Why a menu beats an interview

An open prompt is the enemy of a good draft. Ask a model to "write a post" and it reaches for the generic middle, because nothing roots it to anything real. Google's Vertex AI team describes [grounding](https://cloud.google.com/blog/products/ai-machine-learning/how-vertex-ai-grounding-helps-build-more-reliable-models) as giving a model external information to root its response in reality, which is what cuts down on vague or invented output. A menu of specific anchors is grounding applied to ideation: instead of asking you to supply the specificity, the tool proposes it. One of the richest sources of specifics you already have is your own recent work, and on a machine running Claude Code, that history is sitting on disk.

## Read the session logs to find recent work

Claude Code stores each session as a JSONL transcript. The [docs](https://code.claude.com/docs/en/sessions) put it at `~/.claude/projects/<project>/<session-id>.jsonl`, where `<project>` is your working directory with non-alphanumeric characters replaced by dashes. That slug is lossy: you cannot reliably turn `-Users-nate-localrepo-my-app` back into `/Users/nate/localrepo/my.app`, because both a dash and a dot collapse to the same character. So do not parse the directory name. Each line of the transcript is a full JSON object ([JSON Lines](https://jsonlines.org/)), and the real, unambiguous working directory is a field inside those objects. Read the file and pull it out:

```python
# discover.py — find the real cwd of recent Claude Code sessions
import json, pathlib

SESSIONS = pathlib.Path.home() / ".claude" / "projects"

def session_cwd(jsonl: pathlib.Path) -> str | None:
    """Return the authoritative working directory recorded in a transcript."""
    try:
        with jsonl.open(encoding="utf-8") as f:
            for line in f:
                line = line.strip()
                if not line:
                    continue
                try:
                    obj = json.loads(line)
                except json.JSONDecodeError:
                    continue          # tolerate a partial last line
                cwd = obj.get("cwd")
                if cwd:
                    return cwd
    except OSError:
        return None
    return None
```

The defensive parsing is not optional. The docs warn that the [entry format is internal and changes between versions](https://code.claude.com/docs/en/sessions), so any field can move or vanish on an upgrade. Reading one known field and skipping anything you cannot parse means a format change degrades to "found nothing" instead of a crash.

## Dedupe by real path, then attach a last commit

One repo usually has many sessions, so collect the newest transcript per directory, keep only real git repos, and skip temp dirs. Then attach each repo's last commit so the menu item carries a concrete hook:

```python
import subprocess

def recent_repos(limit: int = 5) -> list[dict]:
    newest: dict[str, float] = {}
    for jsonl in SESSIONS.rglob("*.jsonl"):
        cwd = session_cwd(jsonl)
        if not cwd:
            continue
        path = pathlib.Path(cwd)
        if "/tmp/" in cwd or not (path / ".git").is_dir():
            continue                                  # real repos only
        mtime = jsonl.stat().st_mtime
        newest[cwd] = max(newest.get(cwd, 0), mtime)  # dedupe by real path

    repos = sorted(newest, key=lambda c: newest[c], reverse=True)[:limit]
    out = []
    for cwd in repos:
        last = subprocess.run(
            ["git", "-C", cwd, "log", "-1", "--format=%s"],
            capture_output=True, text=True,
        ).stdout.strip()
        out.append({"path": cwd, "name": pathlib.Path(cwd).name, "last_commit": last})
    return out
```

## Use it as the anchor

Now the opener is a list of real options, not a question. Each project becomes a menu item whose label is the repo and whose subtitle is what you last shipped there:

```python
for repo in recent_repos():
    print(f"{repo['name']}: {repo['last_commit']}")
# local-fitness: release: 0.14.0 — chart calendar style
# claude-skills: feat(ghostwriter): three-tier regression eval suite
```

The user picks one, and that repo plus its recent commits becomes the grounding the draft is written from. No interview, because the specificity came from disk. The same shape powers the radar lane, where the anchors are pre-researched industry items instead of repos, and an evergreen lane for when neither fits.

## Verify it, and the failure mode to expect

The one real failure mode is the format drift the docs call out. Test it by feeding the parser a transcript with a missing `cwd`, a blank line, and a truncated final line, and assert it returns `None` or skips cleanly rather than throwing:

```python
def test_tolerates_bad_lines(tmp_path):
    f = tmp_path / "s.jsonl"
    f.write_text('\n{"type":"meta"}\n{"cwd":"/home/n/app"}\n{"broken', encoding="utf-8")
    assert session_cwd(f) == "/home/n/app"
```

When a Claude Code update reshapes the transcript, this is the test that fails first, in your own suite, instead of the discovery step silently returning an empty menu in front of a user. Read the authoritative field, tolerate the rest, and let the menu degrade quietly when the format moves.
