## What I Built

The honest motivator: my recent posts were getting almost no reach. So I stopped guessing and researched how the LinkedIn algorithm actually works in 2026, then turned the findings into `voice/algorithm.md` — a sourced playbook drawing on Richard van der Blom's Algorithm InSights, Sprout Social, AuthoredUp, and Hootsuite. The skill now reads it on every draft, and the rules are baked into generation and publishing: hook in the first ~210 characters (all that shows before "see more"), keep posts in the ~900–1,500 character range, optimize for *saves* rather than likes, never put an external link in the post body, and run a "golden hour" engagement routine right after posting. I reconciled all of it with my existing voice notes so it chases saves and genuine discussion, never engagement bait.

Next I added a `flow` card type for clean architecture and pipeline diagrams — color-coded stage chips connected by simple chevrons, with the byline pinned at the top so a feed crop can't remove it. It's meant to replace Mermaid for architecture posts, which tends to render as a skinny, unreadable strip on a phone.

The big one was carousels. Multi-slide documents are the highest-reach native format on LinkedIn, so I built support for them end to end: a carousel slide template, a `render_carousel.py` that screenshots each slide into preview PNGs and stitches them into a single PDF, and a `--document` flag on the publisher that uploads that PDF through LinkedIn's documents API. LinkedIn renders the PDF as a swipeable carousel. I verified the whole path live against the real API rather than trusting a dry run.

Throughout, I kept treating the skill as code: 121 tests passing at 100% coverage (including new tests for the renderer and the document-upload path), shellcheck clean, and the SKILL.md still scoring full marks. I also ported the new styles into the shipped example brand guide so a fresh install renders them, and cut the release as v0.3.0.

## What's Next

Two threads worth pulling: automating the "link in the first comment" step so reach-suppressing links never land in the body, and experimenting with portrait (4:5) slide sizing to claim more vertical space in the feed.
