## What I Built

Extended the LinkedIn ghostwriter so a post can optionally carry a visual. There are two kinds: technical diagrams written in Mermaid (flows, architecture, sequence) and designed cards built from HTML/CSS for a single punchy idea. Both render locally through a headless Chromium via Playwright into a high-DPI PNG, so nothing gets sent to a third-party renderer.

The whole thing is opt-in. Text-only posts are still the default, and an image only gets attached after I approve the rendered PNG. On the publish side, the poster now does the LinkedIn image-upload flow — initialize the upload, PUT the bytes, attach the returned image URN — only when an image is actually provided, so the text-only path stays byte-for-byte unchanged.

The second piece is a per-user brand guide. The styling and byline live in a gitignored `diagram.css` so my personal look isn't baked into the shared repo — anyone who clones it starts from a neutral template instead. The byline is a CSS variable that auto-applies to the bottom of every card and diagram, so I set it once rather than retyping it on each visual.

## What's Next
