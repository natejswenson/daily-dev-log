## What I Built

Today I went from "the App Store version is outdated and v2 is half-built on a feature branch" to v2.0 tagged on main and ready for resubmit. The day was structured as one giant pre-launch sprint.

I started with an extremely deep audit — six parallel agents combing the codebase across UX, monetization, onboarding, post-run, visual system, performance, and dead code. The recon doc revealed that documented features I'd been mentally tracking (50 levels, sections, daily challenge, tutorial flow) were actually dead code — the live game is a single-screen endurance reaction loop. That insight alone reshaped the polish scope: stop trying to re-light dead surfaces, ship the one screen at world-class quality.

From there I ran the polish pass: AppFont meta tokens to fix three contrast offenders against WCAG AA, leaderboard chart icon promoted from invisible to discoverable with a first-session pulse, ShareManager wired into the death reveal with the SHARE pill gated to runs worth sharing (depth ≥ 10 or new best), DeathReveal tap-to-dismiss with a 6.5s floor and 200ms post-dismiss cooldown to stop dismiss-taps from bleeding into a fresh run start, and a chunky onboarding cascade that teaches "land at 1.000s" in the first sixty seconds without ever feeling like a tutorial.

Compliance work was the unsexy must-do: UMP/CMP for EEA consent on launch (running in parallel with auth restore), ATT prompt at first-run-end (Apple HIG says never on cold launch), and a hard guard that force-disables rewarded ad serving in Release if the unit ID is still pointed at Google's test sentinel — protects against shipping test ads to real users by accident. The interstitial logic got a first-session grace gate and a tightened foreground-return threshold so people don't see an ad on every app switch.

Audio: the milestone chime at 5/10/15 was sounding *funereal* — descending pitches, low amplitude, heavy reverb. Replaced with an ascending C5→E5→G5→C6→E6 arpeggio at 1.75× the master amplitude with a clean 60ms end-fade. Now it actually feels like a celebration.

Accessibility: VoiceOver labels on all load-bearing UI (orb, trophy, leaderboard icon, death reveal) and Reduce Motion respected across every `.repeatForever` animation in the app — the leaderboard pulse, the orb breath, the death reveal stagger.

Performance: killed a pile of dead state and dead `.repeatForever` animations that were silently scheduling work on every frame. Then did a bulk delete of ~3,000 LOC of orphan code from the dead daily-challenge / level-map era.

Then the App Store rejected the upload twice. First fail: I had dropped iPad target to iPhone-only ("1") to avoid layout work, but the user wanted full iPad. Restored Universal target ("1,2"), generated proper iPad app icon entries (152px, 167px), bumped the orb cap from 520 → 640pt on wide containers so it actually anchors the iPad Pro screen, and anchored UIActivityViewController's popover to the bottom-center of the host view (without `sourceRect` it crashes on iPad). Second fail: Apple requires either all 4 orientations or `UIRequiresFullScreen=true` for iPad multitasking. The game is portrait-only by design, so I opted out of multitasking via `UIRequiresFullScreen` and bumped the build to 4.

Capstone: I cut a 25-second App Store promo video from a simulator screen recording. Brand intro using the actual app icon (not a rendered wordmark — Digital-7 renders 's' as '5' which reads as "1.005"), continuous natural-speed gameplay (no jarring 3× speedups), instrumentation-style monospaced captions matching the in-game `±232ms` register, and a brand outro. 886×1920 H.264, ready for App Store Connect.

Tagged `v2.0` and pushed to main.
