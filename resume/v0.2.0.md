## What Shipped

The headline is that resume now has a benchmark you can trust, and it costs nothing to run.

`npm run benchmark` runs the pipeline against 7 real jobs and scores the output. The judges run on the CLI, so there is no API bill to run it, and the gating only looks at the treatment so a noisy baseline cannot move the verdict. That gives a repeatable read on quality instead of eyeballing a tailored resume and hoping it got better.

The other half is speed. The retry path used to redo more work than it needed to. Now it retries only the summary, which takes the slow case from about 39 seconds down to about 5.3 seconds. Same output, far less waiting. It also stopped emitting no-op bullets, so a retry no longer pads the resume with lines that say nothing.

All of it is quality-gated and the benchmark is CI-portable, so this can run on the same path as the rest of the skills rather than only on my machine.

## What's Next

Leaning on the benchmark to catch quality regressions automatically, and using it to justify the next round of speed work rather than guessing where the time goes.
