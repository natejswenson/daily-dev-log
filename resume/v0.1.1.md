## What I Built

OneTap Resume started as a web app that tailors a résumé to a specific job posting and renders a polished PDF. Today I turned its pipeline into a fully self-contained Claude Code skill — `/onetapresume` — that parses a résumé, pulls the job posting, rewrites the bullets to lead with job-relevant framing (without inventing facts), and renders a multi-template PDF. The whole thing runs locally through the `claude` CLI on your subscription, so there's no API key and no per-run cost.

The headline was performance. A real run was taking five to nine minutes, sometimes hanging outright, and it wasn't obvious why. I instrumented the nested `claude` call and found the smoking gun: a four-bullet résumé produced ~800 characters of JSON but billed **16,000 output tokens**. The missing tokens were *extended thinking* — the model was spending two minutes "thinking" on a rule-dense prompt before emitting a tiny answer. Disabling thinking dropped a call from ~130s to ~10s. I didn't trust the eyeball, so I ran an eval sweep across thinking budgets (0 / 4000 / 8000) and the scorer settled it: zero thinking was both the **fastest and the highest-quality** — more thinking made the output slower *and* worse. Net result on a real 26-bullet résumé: roughly **13× faster**. Along the way I also caught that the default model (Sonnet, uncached on the subscription path) was timing out 3/3, so the CLI now defaults to Haiku.

To replace the self-audit the prompt used to do "out loud" in thinking tokens, I moved those checks into deterministic code — banned summary phrases, scope qualifiers not present in the source, derived "X years" durations, invented numbers — with one targeted corrective retry. Same guardrails, none of the latency.

Then the UX. Tailoring now streams live progress (a spinner, elapsed timer, and token count that degrade to clean one-line-per-phase output when piped), and results print as proper tables. I added a native macOS file picker so you can select your résumé instead of typing a path, and — the fun part — an interactive **style picker**: tailoring runs once, then switching between the seven templates is a ~0.5s re-render with the PDF re-opening as a live preview, looping until you save. It signs off with a "Thanks for using OneTap Resume!".

Finally, I made it ready for other people to actually use. I swept ~1,580 lines of dead and vestigial code, dropped an unused dependency that pulled in 92 transitive packages, fixed a footgun where the skill would silently bill a paid API key if one happened to be in the environment, added an MIT license and full packaging metadata, and rewrote the README for outside users. I wired up a release workflow that tags and publishes a GitHub Release automatically on merge to `master`, then shipped **v0.1.0** and a quick **v0.1.1** to get the license into a tagged release.

## What's Next

Push tailoring under a minute by trimming the per-bullet before/after payload, and add an eval-gated quality check so future prompt changes can't regress the output.
