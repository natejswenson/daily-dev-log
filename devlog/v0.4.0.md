## Shipped

devlog 0.4.0 changes what `/devlog` writes. Each release entry used to be a narrative summary of what shipped. Now it is a researched, end-to-end implementation guide: a short "what shipped" hook, then a deep dive that teaches the engineering topic the work touched, with copy-paste code, a `## Sources` section, and inline citations. A new `deepDive` config block sets the topic domains the post may explore and a `minSources` floor for how many reputable sources it must cite.

The instructions are the easy half. The half worth writing about is trust. The moment you ask a model to "cite reputable sources," you have to assume it will sometimes invent them, so the useful work is encoding a citation floor and a structure as a contract, then checking the output against that contract instead of trusting the prose.

## Why "cite your sources" is not enough

A skill is a Markdown spec the model reads and follows; Anthropic's own Agent Skills are folders whose `SKILL.md` is deliberately both machine-parseable and human-readable ([Anthropic: Equipping agents with Agent Skills](https://www.anthropic.com/engineering/equipping-agents-for-the-real-world-with-agent-skills)). That makes the behavior easy to change and impossible to guarantee. An instruction is a strong suggestion, not an enforced invariant.

Citations are exactly where that gap bites. Large-scale analysis of references in published papers found fabricated or invalid citations rising sharply as model-assisted writing spread ([GhostCite: citation validity in the age of LLMs](https://arxiv.org/abs/2602.06718)), and models are often unable to tell when a reference they produced is real ([Do Language Models Know When They're Hallucinating References?](https://arxiv.org/abs/2305.18248)). A confident, well-formatted citation to a paper or doc that does not exist is the default failure mode, not an edge case. So a post that merely *asks* for sources can read as rigorous while linking to nothing real.

The fix is the same discipline Anthropic recommends for agents generally: define what good looks like, then measure it in the loop rather than eyeballing the output once ([Anthropic: Demystifying evals for AI agents](https://www.anthropic.com/engineering/demystifying-evals-for-ai-agents)). For a writing skill, "what good looks like" is a structural contract, and "measure it" is a linter you can run on every generated post.

## Setup: declare the contract as config

Start by making the floor data, not prose. The release added a `deepDive` block so the citation minimum and the allowed topic domains live in config, where they can be read by both the skill and a checker:

```json
{
  "deepDive": {
    "topicDomains": ["AI", "DevOps/SRE", "software engineering"],
    "minSources": 3
  }
}
```

The contract itself is small and explicit: every post must have a `## Shipped` hook, at least one language-tagged code block, a `## Sources` section carrying at least `minSources` resolvable links, and a `## Changelog`. Those are the load-bearing parts a reader relies on, so they are the parts worth enforcing.

## Build: lint the post against the contract

The checker reads a finished post and returns a list of problems. It is deliberately boring: parse the text, look for the required sections, count the citations under `## Sources`, and confirm at least one real code block. Boring is the point, because a deterministic check cannot be talked out of failing the way a model can.

```python
# lint_post.py
import re, sys, pathlib

REQUIRED = ("Shipped", "Sources", "Changelog")
LINK = re.compile(r"\[[^\]]+\]\((https?://[^)]+)\)")
CODE_BLOCK = re.compile(r"^```[a-z]+\n.*?\n```", re.S | re.M)

def section(text: str, heading: str) -> str:
    """Return the body under a real `## heading` LINE, up to the next `## `.
    Anchoring on a line start matters: the post mentions `## Sources` inline in
    its own prose, and a naive text.split() would match that, not the section."""
    start = re.search(rf"(?m)^## {re.escape(heading)}\s*$", text)
    if not start:
        return ""
    rest = text[start.end():]
    nxt = re.search(r"(?m)^## ", rest)
    return rest[: nxt.start()] if nxt else rest

def lint_post(path: str, min_sources: int) -> list[str]:
    text = pathlib.Path(path).read_text(encoding="utf-8")
    problems = [f"missing section: ## {h}" for h in REQUIRED if not section(text, h)]
    if not CODE_BLOCK.search(text):
        problems.append("no language-tagged code block")
    cited = LINK.findall(section(text, "Sources"))
    if len(cited) < min_sources:
        problems.append(f"{len(cited)} cited sources, need >= {min_sources}")
    return problems
```

## Verify: resolve the links, do not just count them

Counting links is not enough, because a fabricated citation is still a syntactically valid Markdown link. The check that actually catches hallucinated references is resolving each URL, since a made-up source usually fails to resolve or 404s:

```python
# check_links.py
import urllib.request

def dead_links(urls: list[str], timeout: float = 10.0) -> list[str]:
    """A fabricated citation can still look like a valid link. Resolve each one;
    report the URLs that do not come back."""
    dead = []
    for url in urls:
        req = urllib.request.Request(
            url, method="HEAD", headers={"User-Agent": "devlog-linter"}
        )
        try:
            with urllib.request.urlopen(req, timeout=timeout) as resp:
                if resp.status >= 400:
                    dead.append(f"{url} -> HTTP {resp.status}")
        except Exception as exc:
            dead.append(f"{url} -> {type(exc).__name__}")
    return dead
```

Then wire both into the pipeline so no post ships unchecked. Running it on a draft prints the contract violations and stops the build on any of them:

```bash
$ python lint_post.py devlog/v0.4.0.md
2 cited sources, need >= 3
$ # fix the post, then it exits clean and the link resolver runs in CI
```

A HEAD request is best-effort, not proof: a live URL can still point at the wrong content, and a flaky host can fail a real link, so treat resolution as a strong signal you review, not an oracle. It catches the common, embarrassing case, which is a link to something that was never there. The deeper lesson is the one the eval guidance keeps making: the model is a generator, and the trustworthy part of the system is the check you run on what it generated, not the generation itself.

## Next

The contract lives in the skill's instructions today, and the linter is the deterministic backstop. The natural next step is to run that linter as a gate in the publishing path, so a post that drops below the citation floor or loses a section cannot be pushed, the same way tests gate code.
