## What I Built

The drop-in React component for rendering a dev log had a real accessibility gap. Each entry's expand/collapse was a bare `onClick` on the `<article>` element, with no role, tabIndex, `aria-expanded`, or keyboard handler. Since the README points people to copy that component straight into their own site, anyone who did inherited a feed that keyboard and screen-reader users couldn't operate.

I reworked the toggle into a proper disclosure control on the entry header: `role="button"` with tabIndex, `aria-expanded` and `aria-controls` for screen-reader semantics, and an Enter/Space handler that prevents default on Space so the page doesn't scroll. I also added a `:focus-visible` outline so keyboard focus is actually visible. Moving the toggle to the header instead of the whole card keeps the expanded body's links out of an interactive ancestor and restores normal text selection. The visuals are unchanged; padding, hover, and cursor just moved from the entry to the header.

Then I cut release 0.1.9 so anyone pulling the package gets the accessible version by default.

## What's Next

Sweep the rest of the example app for similar gaps and fold the disclosure pattern into the docs so it's the default people start from.
