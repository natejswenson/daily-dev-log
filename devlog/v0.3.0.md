## What Shipped

devlog used to summarize each day's commits. Now it summarizes each release. When I tag a version, devlog writes one entry for that release and pushes it to the dev-log repo, and re-running does nothing until the next tag. The log is one clean entry per version instead of a daily stream.

The bigger change is voice. Entries now come from a voice profile instead of a generic standup tone, so the log actually sounds like me. The profile is configurable with a `voicePath`, and it falls back to my ghostwriter voice or a bundled default. It applies my tone and my "never do" rules, and it ignores the LinkedIn reach tuning, since a dev log is not a feed.

To make it hold up across a monorepo, each project sets its own tag prefix, so several skills in one repo each track their own releases. Only real version tags become entries; prerelease and build-metadata tags are skipped. Tag names are validated as untrusted input before they ever touch a shell.

## What's Next

I want to see how these voice-written entries read across the next few releases, then tighten the template from what those show.
